# Microsoft Xbox 360 Data Storage Guide

## Console Overview
- **Release Dates**: North America (November 22, 2005), Europe (December 2, 2005), Japan (December 10, 2005)
- **Active Years**: 2005-2016
- **Regional Variants**:
  - Original (2005): Prone to hardware failures
  - S/Slim (2010): Redesigned, more reliable
  - E (2013): Final revision, simplified design

## Storage Media
- **Disc Capacity**: 6.8GB (XGD2), 7.95GB (XGD3) dual-layer DVDs
- **Internal Storage**: 2.5" SATA HDD (20GB to 500GB)
- **Memory Units**: 64MB, 256MB, 512MB flash storage
- **Digital Distribution**: Xbox Live Arcade, Games on Demand

## Archival Storage
### Recommended Formats
- **.iso**: Standard disc image format
- **.god**: Games on Demand container format
- **.xex**: Xbox executable format

### Best Practices
- Preserve disc images with complete data
- Archive Xbox Live Arcade titles
- Include profile and save data
- Document disc format (XGD2 vs XGD3)
- Preserve system updates and dashboard versions

## Emulation Storage
### Current Status
- **Xenia**: Primary emulator in development
- **Compatibility**: Limited but improving
- **Performance**: Requires powerful hardware
- **File Formats**: ISO and extracted files supported

### Considerations
- Complex PowerPC architecture challenges emulation
- Save data stored in profile containers
- Xbox Live features not preserved
- Total library size: 10TB+ uncompressed

## Digital Storage Considerations
- **Space Requirements**: Very high - massive library
- **Backup Strategy**: Critical due to Xbox Live store closure
- **Organization**: Separate physical from digital titles
- **Metadata**: Use scene databases with caution
- **Legal Issues**: Current emulation development
- **Future Planning**: Preservation efforts ongoing
