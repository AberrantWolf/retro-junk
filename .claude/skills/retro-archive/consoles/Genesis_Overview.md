# Sega Genesis/Mega Drive Data Storage Guide

## Console Overview
- **Release Dates**: Japan (October 29, 1988 as Mega Drive), North America (August 14, 1989 as Genesis), Europe (1990)
- **Active Years**: 1988-1997
- **Regional Variants**:
  - Mega Drive (Japan/Europe): Original name and design
  - Genesis (North America): Renamed due to trademark issues
  - Various model revisions with different audio chips

## Storage Media
- **Cartridge Capacity**: 512KB to 4MB (32 Mbit maximum)
- **Typical Sizes**: 512KB-2MB for most games
- **Save Storage**: Battery-backed SRAM, EEPROM, or password systems
- **Add-ons**: Sega CD (650MB), 32X enhancement

## Archival Storage
### Recommended Formats
- **.bin**: Raw binary ROM dump
- **.md**: Mega Drive format
- **.gen**: Genesis format (identical to .bin)

### Best Practices
- Preserve regional ROM variations
- Include save data files when applicable
- Archive Sega CD disc images separately
- Document cartridge mapper information
- Preserve 32X ROM dumps

## Emulation Storage
### Recommended Formats
- **.bin**: Most compatible format
- **.md/.gen**: Alternative naming conventions
- **.zip**: Compressed format supported by most emulators

### Considerations
- Good emulation accuracy available
- Save data stored in separate files
- Sega CD requires disc images and BIOS
- 32X games need special emulator support
- Total library size: ~500MB (cartridge games only)

## Digital Storage Considerations
- **Space Requirements**: Low - manageable collection size
- **Backup Strategy**: Easy to preserve completely
- **Organization**: Separate by region and add-on compatibility
- **Metadata**: Use No-Intro or GoodGen databases
- **Compression**: Good space savings with ZIP
- **Emulation**: High-accuracy preservation possible with multiple active emulators
