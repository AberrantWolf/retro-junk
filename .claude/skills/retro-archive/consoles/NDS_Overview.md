# Nintendo DS Data Storage Guide

## Console Overview
- **Release Dates**: North America (November 21, 2004), Japan (December 2, 2004), Europe (March 11, 2005)
- **Active Years**: 2004-2014
- **Regional Variants**:
  - Nintendo DS (2004): Original "phat" model
  - Nintendo DS Lite (2006): Slimmer design, brighter screens
  - Nintendo DSi (2008): Cameras, SD card slot, DSiWare
  - Nintendo DSi XL (2009): Larger screens

## Storage Media
- **Game Card Capacity**: 8MB to 512MB
- **Typical Sizes**: 16MB-256MB for most games
- **Storage Technology**: ROM with integrated flash memory for saves
- **Save Storage**: Integrated EEPROM in game cards
- **DSiWare**: Downloaded games stored on internal memory or SD card

## Archival Storage
### Recommended Formats
- **.nds**: Standard Nintendo DS ROM format
- **.dsi**: DSi-enhanced games
- **.bin**: Raw binary dumps

### Best Practices
- Preserve complete ROM dumps including save areas
- Archive DSiWare titles before eShop closure
- Include save data when applicable
- Document DSi-enhanced features
- Preserve homebrew and flash cart software

## Format Documentation
- [NDS ROM Format](../formats/NDS.md) - Header structure, checksums, game code, detection

## Emulation Storage
### Recommended Formats
- **.nds**: Universal emulator compatibility
- **.zip**: Compressed format for space saving
- **.7z**: Alternative compression

### Considerations
- Dual screens require special emulator layout
- Touch screen functionality essential for many games
- Microphone input supported in some emulators
- Wi-Fi features not preserved in emulation
- Total library size: ~50GB+ (complete set including DSiWare)

## Digital Storage Considerations
- **Space Requirements**: Moderate to high
- **Backup Strategy**: Important for DSiWare preservation
- **Organization**: Separate DS games from DSi-enhanced titles
- **Metadata**: Use No-Intro database for verification
- **Compression**: Significant space savings possible
- **Emulation**: Excellent preservation with DeSmuME and melonDS
