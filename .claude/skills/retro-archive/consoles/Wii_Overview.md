# Nintendo Wii Data Storage Guide

## Console Overview
- **Release Dates**: North America (November 19, 2006), Japan (December 2, 2006), Europe (December 8, 2006)
- **Active Years**: 2006-2017 (production ended 2013)
- **Regional Variants**:
  - Different system menu languages and regions
  - GameCube compatibility varies by model
  - Various colors released over time

## Storage Media
- **Disc Capacity**: 4.7GB single-layer, 8.5GB dual-layer DVDs
- **Disc Format**: Proprietary Wii Optical Disc (WOD)
- **Internal Storage**: 512MB NAND flash memory
- **External Storage**: SD cards, USB drives for saves and channels

## Archival Storage
### Recommended Formats
- **.iso**: Standard disc image format
- **.wbfs**: Wii Backup File System (space-efficient)
- **.rvz**: Dolphin's compressed format

### Best Practices
- Preserve complete disc images including update partitions
- Include NAND dumps for system preservation
- Archive save data and Mii characters
- Document system menu version and region
- Preserve WiiWare and Virtual Console titles

## Emulation Storage
### Recommended Formats
- **.rvz**: Best compression and Dolphin compatibility
- **.iso**: Universal emulator support
- **.wbfs**: Space-efficient but limited compatibility

### Considerations
- Motion controls require special emulator setup
- Save data stored in separate files
- WiiWare titles need separate preservation
- GameCube backwards compatibility in emulation
- Total library size: ~6.56TB uncompressed

## Digital Storage Considerations
- **Space Requirements**: Very high - requires multiple TB storage
- **Backup Strategy**: Critical due to massive library size
- **Organization**: Separate physical games from WiiWare/VC
- **Metadata**: Use Redump database for disc verification
- **Compression**: RVZ essential for practical storage
- **Network**: Consider NAS for large collections
