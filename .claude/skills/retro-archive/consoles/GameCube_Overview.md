# Nintendo GameCube Data Storage Guide

## Console Overview
- **Release Dates**: Japan (September 14, 2001), North America (November 18, 2001), Europe (May 3, 2002)
- **Active Years**: 2001-2007
- **Regional Variants**:
  - Different disc regions (NTSC-J, NTSC-U, PAL)
  - Various case colors by region
  - Game Boy Player attachment compatibility

## Storage Media
- **Disc Capacity**: 1.5GB mini-DVDs
- **Disc Format**: Proprietary Nintendo Optical Disc (NOD)
- **Save Storage**: Memory Cards (59 blocks standard, 1019 blocks available)
- **Disc Protection**: Custom format prevents standard DVD reading

## Archival Storage
### Recommended Formats
- **.iso**: Standard disc image format
- **.gcm**: GameCube-specific format (identical to ISO)
- **.rvz**: Dolphin's compressed format for long-term storage

### Best Practices
- Preserve full disc images including unused sectors
- Include memory card saves (.gci or .raw formats)
- Document disc region and revision information
- Maintain separate collections by region
- Archive Game Boy Player BIOS when applicable

## Emulation Storage
### Recommended Formats
- **.rvz**: Dolphin's native compressed format (recommended)
- **.iso**: Universal compatibility
- **.gcm**: Alternative naming convention

### Considerations
- RVZ format offers 30-50% size reduction with no quality loss
- Memory card files stored separately
- Some games require specific Dolphin settings
- Progressive scan and widescreen patches available
- Total library size: ~570GB uncompressed, ~300GB with RVZ

## Digital Storage Considerations
- **Space Requirements**: High - requires substantial storage
- **Backup Strategy**: Essential due to large file sizes
- **Organization**: Separate by region, consider genre folders
- **Metadata**: Use Redump database for verification
- **Compression**: RVZ format highly recommended for space savings
- **Performance**: SSD recommended for optimal loading times
